# 学习方法
1. 要有耐心，不要只想着写代码。要多考虑需求和she ji
2. 抛弃程序员思维
3. 技术永远是为业务服务的，技术是实现业务增长的工具

# 注意事项
1. 不要只关注于细节，要看整体，看范围
2. 设计时判断可行性，不确定就调研一下
3. 设计要考虑复杂度，越简单越好，不要过度设计，不要为了设计而设计
